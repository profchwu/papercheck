# 精簡版 Prompt 彙整

以下僅整理使用者當初提出的 prompt 指令與需求。

## 原始需求

寫一個 HTML 版本的檢查程式，可以讓我上傳摘要和關鍵字，並針對符合「清華大學永續論文」的規範提供修改建議。

包含論文的 title、abstract 與 keywords，請針對需要修改的地方提供建議，並給予紅色的標註。

具有文字框，讓我可以直接把 title、abstract、keyword 一次貼上去，程式會自動區分哪一些是 title、摘要與 keyword，並分別給予整體的建議。

需先上網搜尋清華大學關於「永續論文採認」的資料，並搜尋網路上相關的檢索資料。全部下載之後，再開始寫程式。

先搜尋資料，接著寫一個規劃草案給我，確認之後再開始寫正式內容。

## 規劃書需求

寫出程式規劃書，需列出程式功能與需要參考的資料來源連結與檔案。

英文介面為主，採認風險需要有明確的「通過」或「不通過」。

需要先去下載 Elsevier 符合所有 SDGs 17 項的相關 keyword 文件，才能開始製作。

檢查結果需包含這篇論文比較適合 SDGs 的哪一項主題，並顯示在畫面上。

目前的規劃書檢測出來的結果，是否能夠完全符合清華大學對於永續論文的認定標準？

程式部分必須能夠完全使用前端網頁技術 HTML 來分析執行即可，方便我部署在 GitHub 上面。

前端網頁也必須很精美，然後能夠先依據論文最可能的 SDG 主題，載入對應的 Elsevier 的 keywords，才不會資料爆掉。

這份規劃書必須符合一項要求：讓我能使用其他的 AI 開發工具，並根據這份規劃書寫出程式。

## 程式製作需求

寫出程式給我。

## GitHub 部署需求

可以自動幫我部署在 GitHub 上面嗎？

建立新的 repo。

repo：papercheck

public

直接操作瀏覽器把檔案上傳。

## 欄位解析需求

增加 title、abstract、keyword 的判斷準確性，不要要求那麼嚴格的格式。

通常第一個段落是 Title，第二個段落是 Abstract，第三個部分是 Keyword。

Title、Abstract、Keywords 這三個欄位的格式抓取判定要寬鬆一點，不需要考慮大小寫，以及有沒有一定要打冒號等。

## 標題、作者與免責聲明需求

標題加上「國立清華大學數理教育研究所 吳智鴻教授開發」。

另外加上用英文說明，本判斷結果僅供參考，判定結果還需要以清華大學為準。

標題可以加上修正的版本號，從 V1.0 開始算，並加上修正的日期。

作者用中文名稱：「國立清華大學數理教育研究所 吳智鴻教授開發」。

## 測試需求

檢測我這個範例檢測是否能通過，以及是否能抓到正確的欄位。

然後再測試一次我的範例是否可以正確抓取所有欄位與判斷正確。

是否有正確地檢查 SDG 17 項的精準 keywords？

## 文件需求

寫一個這個程式的說明文件，並說明每一次版本的更新。做成 README.md 並上傳到 GitHub。

另外把程式的規格書也上傳到 GitHub。

把跟你開發這個程式相關的指令 Prompt 整理彙總成過程，並製作成 Prompt.md GitHub。

這些檔案我的本機資料夾也要有一份。

這些文件都必須是中文的，請翻譯成中文後再次上傳。

再寫一個精簡版的 prompt，只需要彙整我當初跟您下的 prompt 指令即可，不要加入其他我沒有說的內容。
